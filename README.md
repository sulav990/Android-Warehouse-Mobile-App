# Android-Warehouse-Mobile-App
Project 3 mobile architect
The Warehouse Inventory Manager was developed to provide small businesses with a secure, persistent tool for tracking stock levels and receiving automated low-stock alerts. To address these needs, I implemented a streamlined UI consisting of a secure Login/Registration screen and a main Inventory Grid that uses a RecyclerView for real-time data display. My design was successful because I prioritized a "data-first" approach, ensuring the SQLite backend was robust before crafting a responsive, high-contrast interface using ConstraintLayout. This ensured that the app remained intuitive and functional, even for users working in fast-paced warehouse environments.

During the development process, I adopted a modular "feature-block" strategy, building and testing individual components like database CRUD operations and SMS notification triggers in isolation. I ensured functionality through iterative manual testing, which was critical for uncovering how permission denials specifically for the SMS feature ,would impact the user experience. I successfully innovated by creating a "graceful degradation" system, where the app remains fully operational even if the user denies certain permissions. I feel I most effectively demonstrated my technical skills in the integration of the SQLite database shell with the UI, creating a seamless bridge between persistent data storage and a user-centered interface.
